# Politique de versions

Les versions initiales reflètent les versions stables disponibles lors de la génération du kit. Pendant SPRINT-00 :

1. vérifier chaque version avec la documentation officielle ;
2. choisir uniquement des versions stables ;
3. générer et committer les lockfiles ;
4. rendre CI et builds verts ;
5. enregistrer toute modification dans le changelog.

Ne pas effectuer de mise à niveau majeure pendant un sprint fonctionnel sans plan d'exécution distinct.
