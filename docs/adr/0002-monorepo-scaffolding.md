# ADR 0002 - Monorepo Scaffolding

Status: Accepted

Date: 2026-06-20

## Context

The project needs a monorepo with separate web, API, domain, contracts, AI, optimization, infrastructure, documentation, and test areas. The repository already uses top-level folders requested by the project owner: `backend`, `frontend`, `python-services`, `infrastructure`, `docs`, and `tests`.

## Decision

Keep the requested top-level folders and map the architecture boundaries inside them.

- Backend lives under `backend/src`.
- Frontend lives under `frontend/web`.
- Python services live under `python-services`.
- Local platform dependencies live under `infrastructure`.
- Cross-cutting tests live under `tests`.

The backend uses a single .NET solution file and separate projects for API, Application, Contracts, and Domain. The solution file uses the `.slnx` format generated by the installed .NET SDK.

## Consequences

Positive:

- The project owner sees the exact high-level folder structure they requested.
- The architecture still keeps module boundaries explicit.
- Sprint 1 can start in the domain project without infrastructure coupling.

Trade-offs:

- The `.slnx` format requires a recent .NET SDK.
- The current skeleton is intentionally thin and does not yet implement domain behavior.
